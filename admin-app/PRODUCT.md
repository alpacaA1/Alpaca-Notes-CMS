# Product

## Register

product

## Users

站点维护者在桌面端集中管理文章、日记、知识点与待读内容，主要任务是快速找到稿件、持续写作、整理元信息并保存到 GitHub 仓库。

## Product Purpose

为 Markdown 内容源提供轻量、可靠的编辑后台。在不引入重型 CMS 的前提下，让内容检索、编辑、预览、图片上传和发布设置形成连贯工作流。

## Brand Personality

克制、专注、可信。界面应像成熟写作工具，减少装饰和干扰，让正文始终是视觉与交互中心。

## Anti-references

避免重型 CMS 的多栏常驻布局、重复操作入口、装饰性仪表盘、过度玻璃质感，以及为了展示功能而挤压正文画布的侧栏。

## Design Principles

- 正文优先，辅助信息按需出现。
- 熟悉的编辑器交互优先于新奇控件。
- 一个任务只保留一个清晰的主入口。
- 保留 Markdown 与仓库作为内容事实源。
- 改动状态、保存结果和危险操作必须清晰可见。

## Accessibility & Inclusion

保持键盘可操作、可见焦点、语义化标签和足够对比度；抽屉与菜单支持 Escape 关闭，并尊重 reduced-motion 设置。
