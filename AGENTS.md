# alpacaA1.github.io 项目规则

## 项目定位
- 这是 Hexo + GitHub Pages 个人主页 / 博客项目。
- `admin-app` 是自定义管理后台，主要用于文章编辑、待读内容、图片上传等内容管理能力。
- Markdown 文件和仓库内容仍然是内容源，不要把项目改造成重型 CMS，除非用户明确改变方向。

## 当前 admin-app 编辑器基线
- 当前编辑器基线是 markdown-only editor with image upload。
- 上传图片存储在 `source/images/YYYY/MM/...`。
- 写入 Markdown 的图片路径必须使用 `/Alpaca-Notes-CMS/images/...`。
- 不要写成 `/images/...`，否则 GitHub Pages 子路径部署下，文章重新打开后图片会丢失。
- 如果未来出现“图片上传后当次可见、刷新或重开文章后消失”，优先检查 Markdown 中持久化的图片 URL。

## admin-app 工作方式
- 对 admin-app、编辑器、内容管理相关任务，默认快速进入实现，不做长时间方案讨论。
- 用户偏好：直接做、少问、快速验证、尽快推上线。
- 修线上问题时，优先快速复现、定位根因、最小修复、构建验证、推送上线。
- 测试以聚焦验证为主，不主动扩大成复杂测试矩阵，除非任务本身要求。
- 不要主动重构无关模块。

## UI 范围控制
- admin editor UI 改动默认限定在登录后的编辑器页面。
- 不主动重设计登录页、站点前台或无关流程。
- 沉浸/预览能力只保留一个清晰入口，不要叠加多个 focus / expand 控件。
- 聚焦编辑或预览时，可以隐藏当前文档框、侧栏等周边模块，让编辑/预览画布最大化。
- 常规编辑态需要保持编辑区和侧边模块的视觉对齐。

## read-later 当前基线
- read-later 在编辑器中保持阅读视图，不再暴露 raw Markdown 模式切换。
- 打开 read-later 条目时，左侧归档列表切换为紧凑阅读目录。
- 左侧阅读目录包含返回归档、内容目录、回到顶部等轻量能力。
- 右侧保持统一侧栏，包含 `信息` / `评论` tabs。
- 评论 tab 编辑结构化字段：`原文摘录`、`我的总结`、`我的评论`。
- 存储格式仍然是 Markdown body。
